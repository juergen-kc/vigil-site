Getting Vigil

How do I install the visionOS version?

Vigil is on the App Store. On your Apple Vision Pro, open the App Store, search for “Vigil: Situation Room,” and tap to purchase ($4.99) and install — or open the listing directly at apps.apple.com/app/id6773060534.

How do I install the macOS version?

Download the latest signed and notarized .dmg from GitHub Releases, drag Vigil into your Applications folder, and launch it. Future updates arrive automatically through Sparkle — the macOS auto-update framework.

Is Vigil a subscription?

No. Vigil is not a subscription. The visionOS App Store version is a one-time $4.99 purchase, with every layer and feature included. The macOS version is free during its public beta. Vigil will never be a subscription.

Troubleshooting

The globe is blank or pins are missing

Most layers fetch data over the network. If you just launched the app, give it 10–20 seconds to populate the first batch. If a specific layer stays empty, open Settings → Layers and confirm the layer is enabled. Some layers (Weather, Calendar) need explicit permission — the system prompt appears the first time you enable them.

News items aren’t appearing

The News layer ships empty. Open Settings → News and add one or more RSS or Atom feed URLs. After the next refresh (every 15 minutes by default), news items will appear in the feed and as pins on the globe if a location can be extracted.

Flights stop updating

Vigil uses OpenSky Network for free flight tracking. OpenSky imposes per-minute rate limits on anonymous clients. If you fly the globe to a busy region, you may briefly exceed the limit; Vigil backs off and resumes automatically. For higher quotas, add OpenSky credentials in Settings → API Keys.

Why aren’t the AI features responding?

On the visionOS App Store version, AI briefings and chat run entirely on-device using Apple’s built-in Foundation Models — no account or API key needed, and they work offline. If responses don’t generate, make sure Apple Intelligence is enabled on your device and try again. (The macOS beta currently focuses on the weather and earthquake layers, with the full set planned for a later update.)

Calendar layer shows nothing

The Calendar layer needs Calendar access. The first time you enable it, the system prompts you to allow access. If you declined, you can re-enable it in System Settings → Privacy & Security → Calendars (macOS) or Settings → Privacy → Calendars (visionOS). Events without a geocodable location are skipped — only events with addresses or coordinates appear as pins.

Ambient mode doesn’t cycle through my watchlist

Ambient mode requires at least one saved watchlist entry to cycle through. Long-press any feature on the globe and choose “Save to watchlist,” then toggle Ambient mode from the toolbar. The globe will rotate to each location in sequence.

FAQ

What data does Vigil send anywhere?

Only what you trigger by enabling a layer or feature. See the full privacy policy for the per-provider breakdown.

Does Vigil sync between my Mac and Vision Pro?

Not yet. Each install keeps its own watchlist and settings locally. The iCloud entitlements are in place to support sync in v1.1; until then, the two installs operate independently.

Can I add my own data layer?

Yes — Vigil exposes a GlobeLayer protocol family in the VigilEngine Swift package. See the layer system docs for the protocol surface and the existing layer implementations as reference.

Why is Vigil source-available rather than open source?

Vigil is shared on GitHub so curious developers can read the code, fork it for learning, and contribute fixes via pull request. The license restricts redistribution of paid builds and forks to the App Store. The full text is in the repo’s LICENSE file.

System requirements

  • macOS Vigil: macOS 14 (Sonoma) or later. Apple Silicon or Intel.
  • visionOS Vigil: Apple Vision Pro running visionOS 26 or later.
  • Network connection for layer data fetches (Vigil is local-first but most data is remote).

Report a bug or request a feature

Email works without an account. For a public-facing channel, file an issue on GitHub. Please include your platform (macOS or visionOS), app version, and a brief description of what you were doing when the issue appeared.
